# 버전 범위를 고정한다: 하한은 현재 정상 동작이 확인된 버전, 상한은 다음
# 메이저 직전까지. 이렇게 하면 보안/버그 패치(마이너·패치 업데이트)는 받으면서도
# 갑작스러운 메이저 breaking 릴리스로 매일 돌던 파이프라인이 예고 없이 멈추는 걸
# 막는다. 이전엔 버전 고정이 전혀 없어서 CI가 매 실행마다 최신을 새로 설치했고,
# 그 결과 로컬(yfinance 1.2.0)과 CI(최신)가 서로 다른 버전으로 돌고 있었다.
# 의도적으로 올릴 땐 이 하한을 올린 뒤 로컬·CI에서 한 번 돌려보고 반영할 것.
requests>=2.32,<3
python-dotenv>=1.0,<2
supabase>=2.31,<3
pandas>=2.2,<3
beautifulsoup4>=4.12,<5
# yfinance는 야후 페이지 변경에 맞춰 자주 업데이트되므로(오래된 버전은 오히려
# 스크래핑이 깨질 수 있음) 마이너 업데이트는 열어두되 메이저만 막는다.
# 하한을 1.2로 둔 이유: 1.5+는 curl_cffi>=0.15에 의존하는데 이게 Python 3.10+를
# 요구한다. 로컬 개발 환경(Python 3.9)에서도 설치되게 1.2를 하한으로 잡으면,
# CI(Python 3.11)는 최신(1.5.x)을, 로컬(3.9)은 1.2.x를 각각 설치한다 — 둘 다
# <2 안쪽이라 메이저 breaking은 막힌다. (로컬 Python을 3.11로 올리면 완전히
# 같은 버전으로 맞출 수 있다.)
yfinance>=1.2,<2

# 수집한다(카더라 리포트). 사람 계정 로그인 상태는 파일이 아니라 StringSession
# 문자열로 뽑아 .env의 TELEGRAM_SESSION으로 주입한다. 현재 확인 버전 1.44.
telethon>=1.41,<2
# anthropic — 히어로 카드의 '오늘의 요약'을 Claude(Haiku)로 생성한다
# (generate_daily_summary.py). 현재 확인 버전 0.117.
anthropic>=0.117,<1
